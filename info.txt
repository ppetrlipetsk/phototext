Программа сканирует папку с фотографиями, путь к которой задается в параметрах, считывает из xlsx файла данные,
и вставляет  в фотографию надпись из таблицы xslx. Путь к таблице задается в параметрах. Файлы при считывании
сортируются по алфавиту.

Структура таблицы.
Каждая строка таблицы содержит следующую информацию:
- первый столбец: начальный порядковый номер фотографии в списке файлов фотографий (N1).
  Конечный номер будет рассчитан послое считывания порядкового номера из следующей строки (N2). Количество фотографий
  с надписью из текущей строки, будет рассчитано как: N=N2-N1. Если строка последняя, то значением N2 будет количество
  файлов, считанных из заданной папки.
- второй столбец: содержание надписи.

Параметры программы задаются в файле: applicationcontext.xml.
 <bean id="imageEngine" class="com.ppsdevelopment.imagelib.ImageEngine">
dx-смещение надписи по оси x.
dy-смещение надписи по оси y.
withRatio- определяет ширину блока надписи, которая рассчитывается, (как ширина изображения)*widthRatio (по умолчанию 0,67).
heightRatio- определяет высоту блока надписи, которая рассчитывается, (как высота изображения)*heightRatio (по умолчанию 0,1).
fontLineHeightRatio- определяет высоту строки надписи, исходя из которой рассчитывается размер шрифта по формуле:
                     (высота изображения)*fontLineHeightRatio. По умолчанию: 0.2
destinationPath- путь к папке с обработанными изображениями. Он не должен совпадать с sourcePath.
.
.
.

</bean>
<bean id="filesPathReader" class="com.ppsdevelopment.FilesPathReader">
        <constructor-arg index="0" value="[путь к папке с изображениями]"/>
        <constructor-arg index="1" value="[расширение файлов с изображениями]"/>
</bean>
<bean id="importProcessor" class="com.ppsdevelopment.ImportProcessor">
        <property name="filePath" value="[путь к XLSX файлу]"/>
</bean>

При вызове приложения, нужно задать параметр <путь к файлу applicationcontext.xml>. Пример вызова:
java -jar phototext.jar ..//applicationcontext.xml
